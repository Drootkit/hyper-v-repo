当前目录，是用户学习  HyperV 漏洞研究的主目录。

# 注意

当你根据用户的需求，增加或者减少了目录，则需要实时更新该文件中的目录说明。

各种内容记录，必须遵循目录内容安排，禁止随意生成任何文件。

# 目录结构安排如下：

主目录：

- 记录readme
- 存放一些IDE工具的配置信息
- 记录下面列出的各种子目录

base目录：

- 存放基础信息的学习资料，笔记等
- 存放学习路径设计等

utils目录：

- 存放的是一些独立的小工具项目或者代码

misc目录：

- 存放收集到的一些还没有阅读的文章
- 存放一些还没有用到的项目等

hyperv docs目录：

- 存放微软官方对于hyper的介绍等

以CVE名称（漏洞名称）命名的目录（每个漏洞一个独立的目录）：

- 存放该漏洞的学习笔记
- 该漏洞用到的相关环境、工具等

# 提交规范

git commit 格式

[日期] [提交次数(按照总次数累计)] [简要描述当前版本改动，通过逗号隔开]

示例："0426 第一次提交 新增仓库，添加hyper-v基础知识学习记录，以及roocode中的两个skill。"
